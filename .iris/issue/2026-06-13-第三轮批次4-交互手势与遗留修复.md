## 实施记录

2026-06-13 完成，typecheck + 43 单测全绿。

### I — 删除文件手势

- 共享右键菜单（左栏 / 裸树 / issue 表 / SimpleList 共用的 DocContextMenu）末尾新增红色「删除文件…」项。
- 新组件 DeleteDocDialog（模块级 open 模式，挂在 App 根）：永远确认（删除不可逆）；有挂载终端时列出会话名并给两分支——「关闭这些终端并删除」/「重新锚定到项目根并删除」。
- 新指令链：`doc.delete`（与 `doc.save` 同资源 `doc:{path}` 串行，排队中的保存不会与 unlink 竞态；主进程侧限定只删 `.iris/` 下的 `.md`）+ `session.reanchor`（Marina 锚定模型复用：只改 `info.docPath`、PTY 不动，广播 state-changed patch）。
- 删除后的树 / 编辑器 / 选中态清理不写一行新代码——走 chokidar unlink 既有链路（与外部删除同路径）。
- 注意：重新锚定不改变活进程的 `FOCUS_DOC` 环境变量（进程活着改不了）——协议明文允许这种降级（"没被读，人补一句话就行"），终端里的 agent 换聚焦说一句即可。

### J — 编辑器右键菜单

- CrepeEditor 的宿主 div 包上共享 ContextMenu：剪切 / 复制 / 粘贴 / 全选。
- 动作走新 UI helper 通道 `window:edit-action` → 主进程 `webContents.cut()/copy()/paste()/selectAll()`（白名单分发）——渲染端 `execCommand('paste')` 是特权操作拿不到剪贴板，webContents 路径才有完整粘贴保真（含 ProseMirror 的富文本粘贴处理）。
- 焦点处理：菜单关闭后先把焦点还给 `.ProseMirror`（DOM 选区在焦点往返后保留），再触发动作。

### F-2 — portable GPU cache 报错

- 根因定位：双开实例共享同一 Chromium 用户配置目录（userData），第二个实例抢 GPU/disk cache 锁失败报 "Unable to create cache"。portable 包双击两次最易触发。
- 修复：`app.requestSingleInstanceLock()`——拿不到锁的实例直接退出，并让既有实例窗口 restore + focus（`second-instance` 事件）。多实例多项目场景 v1 本就不支持（"一次打开一个项目"），单实例锁与定义书一致。
