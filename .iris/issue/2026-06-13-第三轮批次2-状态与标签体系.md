## 第一轮验收反馈与重做（2026-06-13）

用户两条批注，指出第一版方向不对：① "不只是变色，是精心设计的标签组件"；② "标签应该有预设模板，把对应的匹配字符串填进去，命中就用该模板显示"。即：要的是**模板系统**（设计好的样式预设）+ 把字符串指派到模板，不是"一行一字符串配一个颜色"。

重做（typecheck / 43 单测 / build / 冒烟全绿）：

- **数据模型升级**（`shared/style-maps.ts`）：颜色 id → **模板 id**。新增 `BadgeVariant`（solid 实心 / soft 柔光 / outline 描边 / dot 圆点）× `BadgeColor`（七色），`BadgeTemplate { id, label, variant, color }`，预设库 `BADGE_TEMPLATES`（curated gallery，id 形如 `soft:pine`，label 如「柔光·松绿」），`templateById` 容错解析（未知 → `DEFAULT_TEMPLATE_ID = soft:gray`）。`StyleMaps` 的值改存模板 id；`DEFAULT_STYLE_MAPS` 给六态/两态各配了视觉区分的模板（Done = 实心松绿、Todo = 柔光灰、Canceled = 描边灰…）。`sanitizeStyleMaps` 带旧值迁移（裸颜色名 `foam` → `soft:foam`），所以已落盘的旧 styles.json 读取即自动升级。
- **精致组件**（新 `ui/badge.tsx`）：一个 Badge 组件按模板渲染四变体——solid 实心填充 + base 色文字、soft 柔光底 + 前导色点、outline 发丝描边、dot 色点 + 朴素文字。颜色走内联 CSS 变量（`color-mix` 调透明度），避开 Tailwind 动态类构建期问题，任意七色都生效；gray 落到 `--rp-muted`。圆点 / chevron / 移除 × 内建。StatusBadge、LabelChip 改为薄封装这个 Badge + 查表。删除了旧的 `styleDotVar` / `BADGE_STYLE_IDS` / FNV 配色残留。
- **设置页样式区重做**：每行不再是颜色 swatch，而是 **TemplatePicker**——下拉里用**该行自己的字符串**实时预览全部预设模板（所见即所得地选模板），触发器显示当前模板预览。两张表（状态 / 标签）共用，标题改「→ 模板」。快速添加候选 = 规约状态集 / 项目在用标签；新增字符串默认 `soft:gray`。
- 本项目 `.iris/styles.json` 升级为模板 id 写法，并给 `round-3`（soft:iris）、`bug`（soft:love）配了示例。

用户当时跳过的几项（改色即时生效 / styles.json 热更新）链路未变，重做后仍成立，待重测。
