软件左上角有一个iris标记，然后下面一点还有一个，然后要是打开了iris文件夹，再下面一点还有一个，然后就是中间的这个，是用来做什么的？是不是可以去掉的

## 诊断：三个 "iris" 各是什么

从上到下：

1. **标题栏品牌** `Iris` —— `src/renderer/components/layout/TitleBar.tsx:183`（`text-primary`）。应用名，永远在。旁边还有项目根路径面包屑 `— E:\projects\iris`（`ProjectCrumb`，同文件 36–44 行）。
2. **左栏头部项目名** `iris` —— `src/renderer/components/layout/LeftPane.tsx:64`（`scan.projectName`）。这一行右侧挂着工具条（打开文件夹 / 排序 / 筛选 / 待办 / 新建工作区 / 裸树切换）。
3. **透镜树根节点** `iris`（带 `FolderRoot` 图标）—— `src/renderer/components/lens/LensTree.tsx:252` `RootNodeRow`。可点：选中项目根 → 终端独占全宽，根会话列在其下。是主页重设计的承重节点。

撞成"三个"是因为本项目文件夹正好叫 `iris`，跟 #1 品牌同名。换个项目（如 `my-app`）会看到 `Iris / my-app / my-app`——#1 不再撞，但 **#2 与 #3 这对真冗余仍在**：两者都在显示同一个项目名，上下紧挨。

结论：**中间的 #2 文字可去**。#1 是品牌不动；#3 有交互（终端枢纽）不能去；#2 一旦项目打开，文字就是把正下方 #3 的名字抄一遍——那一行真正有用的是右侧工具条，不是这串名字。

## 方案：左栏头部降级为纯工具条（保留全部功能）

核心：把 #2 从"项目名 + 工具条"降为"工具条"。项目身份在侧栏内只由 #3 根节点承载；全局身份仍由标题栏品牌 + 路径面包屑（#1 区）常驻——它钉在顶栏、不随树滚动消失，所以不丢上下文。

### 改动点（`LeftPane.tsx` 头部 62–177 行）

* 删除项目名 `<span>{scan?.projectName ?? '未打开项目'}</span>`（63–65 行）。

* 工具条容器去掉 `ml-auto`，按钮改左对齐，使该行成为一条紧凑图标条，不再左空一截。

* 「打开项目文件夹」按钮（始终存在）留在最左，作为该行的稳定锚点。

* 其余按钮（排序 / 筛选 / 待办 / 新建工作区 / 裸树切换）逻辑与门控（`scan?.hasIris`）完全不变——功能零删减。

### 边角：项目名在哪些状态下仍需露出

\#3 根节点只在「透镜树 + 已打开 + 有 .iris」时在场。其余状态它不在，需确认身份不丢：

* **未打开 / 打开中 / 出错 / 无 .iris**：左栏正文已有 EmptyState 文案（"打开一个项目开始" / "该项目还没有 .iris/" 等），身份与引导不依赖头部文字——可安全删。

* **裸文件树模式**：`RawTree` 根节点显示的是 `.iris`（`RawTree.tsx`，根 `node.name`），不是项目名；但顶栏路径面包屑仍在——身份不丢。

判定式（如需保险回落）：

```
rootCarriesName = phase === 'ready' && scan?.hasIris && !!scan.root && !rawMode
```

头部文字：`rootCarriesName` 为真则不渲染；否则可选地回落显示项目名。最小实现可直接恒不渲染，靠顶栏面包屑兜底。

### 结果

侧栏纵向从 `Iris / iris / iris` 收敛为 `Iris`（顶栏品牌）→ 图标工具条 → `iris`（可点根节点）：项目名在侧栏只出现一次，品牌与项目身份各归其位，零功能损失。

### 可选连带清理（不在本 issue 必做）

* 顶栏 `ProjectCrumb` 现打印绝对全路径 `— E:\projects\iris`，偏长。可改为只显示项目名或末段路径，让 #1 区更干净。
