### 决策点

- Windows-first：脚本只出 PowerShell 版，hook 命令用 `powershell -NoProfile -ExecutionPolicy Bypass -File`（POSIX 变体留待跨平台里程碑）。
- "经确认"实现为设置页内联两步确认（非系统弹窗）——确认文案明示写入路径与 .bak 备份。
- Claude Code 的 compact 重注入（SessionStart matcher）暂未配置 matcher（默认全部触发源都注入，本身已覆盖 compact 场景）。

## 第一轮验收反馈（2026-06-13，阻断性）

用户实测报错：

```
SessionStart:startup hook  Failed with non-blocking error
status code: （乱码）   C:\Users\liyue\.iris\focus-context.ps1:17 字符: 24
```

根因：脚本含中文与 `——` 破折号，Node 默认写的是**无 BOM 的 UTF-8**，而 Windows PowerShell 5.1 不带 BOM 时按系统 ANSI（中文机 = GBK）解码 `.ps1`，多字节序列被打散、字符串/括号配对错乱，解析在第 17 行 `{` 处崩。错误信息本身也乱码（`����λ��`/`字符`）正是 GBK 控制台 + UTF-8 字节冲突的旁证。

已修（`main/agent-injection.ts`）：

1. `installFocusScript` 写文件时前置 **UTF-8 BOM**（`String.fromCharCode(0xFEFF)`）——PowerShell 5.1 只靠 BOM 识别 UTF-8。
2. 脚本顶部加 `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`——保证注入给 agent 的内容也是 UTF-8 而非乱码。

破折号等非 ASCII 字符保留（有 BOM 后即可正确解析）。**需用户重新「更新脚本」后再开一个 Claude Code 终端复测**：hook 不再报错、首问即知焦点文档。
