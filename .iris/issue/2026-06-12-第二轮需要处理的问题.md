## 细化方案（2026-06-12，Claude 整理，待最终检查）

### 1. 文件名与标题不同步

决议：**文件名即不可变 ID，frontmatter `title` 是唯一显示真相，不做自动同步。**

- 理由：`FOCUS_DOC` 是 spawn 时注入的环境变量，进程活着改不了——重命名会让挂在该文档上的活跃会话锚点全部失效；会话 `docPath`、状态点聚合、chokidar 回声去重都按路径比对；标题是任意字符串而文件名受 Windows 字符/长度限制，双向同步必然有损。
- 展示侧已按 `title ?? 文件名` 工作，无需改动。
- 可选补充：右键菜单加显式"重命名文件以匹配标题"手势（用户主动、确认无活跃会话挂载时），不自动触发。

### 2. 状态下拉菜单只剩一个选项

根因：状态编辑有两个入口。IssuePanel 列表的 StatusCell 是真 DropdownMenu（永远全量四项），没问题；**出问题的是 TypedHeader 用的原生 `<input list>` + `<datalist>`**——原生 datalist 按当前输入值过滤候选，字段已是 `done` 时下拉只剩 `done` 自己，必须先清空才能换。

修复：TypedHeader 换成 combobox（候选永远全量显示 + 保留自由输入），"键是硬的，值是软的"语义不变，只换载体。

### 3. 文档 ↔ 终端联动（按用户修正后）

核心修正：**右栏会话列表按选中文档过滤**，不是全量列表加高亮。

- 触发点：`projectStore.selectDoc`（左栏 lens 树与中栏 issue 列表共用入口）。
- 列表内容：仅显示「选中文档的会话 + 项目根会话」，其它文档的会话不出现。
- 选中文档有会话 → 激活其中最优的一个：状态优先级 active > idle > exited，同级取最近创建（与 `aggregateDocState` 同一套价值观）。
- 没有会话 → 终端区显示挂靠该文档的启动面板（Marina 新建终端面板的对应物）：标题"挂在〈文档名〉"，settings.agents 每项一个按钮，**点击才 spawn，不自动 spawn**。
- "+"下拉保留，作为项目根会话入口。
- 手动点选会话不被打断，仅在下一次 selectDoc 时重新联动；反向联动（点会话选文档）暂不做。
- 实现归属：纯 UI 投影状态，store 层做，不进 ISA 流水线（CQRS：不改变世界）。
- 待定：未选中任何文档时列表显示什么（建议：项目根会话）。

### 4. 右键菜单超出窗口边界

**未复现（用户侧也未复现），挂起。** 备查嫌疑：菜单高度超过窗口可用空间（agents 变多后"用 X 打开"列表变长，需 collisionPadding + maxHeight + 内部滚动）；`zoom-in-95` 进场动画 transform 干扰 Radix 定位测量。待有具体复现位置（哪个菜单、靠哪条边）再处理。

### 5. Marina 终端配置迁移清单

值得迁移（Marina 上已为 Windows 调教过、Iris 场景成立）：

- `appearance.terminalLineHeight` — 终端行高
- `behavior.selectOnCopy` — 划选即复制
- `behavior.terminalRightClick`（menu / paste）— 右键直接粘贴
- `behavior.bracketedPaste` — **第 7 条拖拽粘贴直接依赖，优先搬**
- `advanced.terminalRenderer`（auto / webgl / dom）— WebGL 兼容性逃生舱
- `appearance.uiZoom` — 整体缩放
- `behavior.confirmOnQuit` — 有活跃 PTY 时退出确认

不搬：SSH/remote 系列、`ai.*`（红线：不内嵌 AI）、shell 管理（agents 列表即替代品）、托盘/自启、额外主题（设计语言钉死 Rose Pine 三变体）。

### 6. AI 规约：禁止自作主张建文件

新条款措辞方向：**未经用户明确要求，不创建任何新文件（report 也不例外）；编辑 `$FOCUS_DOC` 所指文件始终允许；修改既有文档的 frontmatter（如 status 流转）允许。** 现行宪法第 4 条（每任务必写 session journal）整条废除：所有新文档一律仅在用户要求时才写，默认不写。

权限边界：改的是 `src/main/iris-templates.ts` 的模板（CONSTITUTION_TEMPLATE + AGENTS_GUIDANCE，管未来新项目）；**本项目已落盘的 `.iris/CONVENTIONS.md` 是用户手写契约，agent 不碰**——由用户自改，或用户明确授权后同步。注意现行宪法第 4 条（每任务必写 report）与新规则冲突，模板修订时一并改掉。

### 7. 拖拽文档到终端粘贴全文

- 拖源：左栏 lens 树 + issue 列表行，`dataTransfer` 放 `application/x-iris-doc`（路径）+ `text/plain` 兜底（裸路径，拖出应用也有意义）。
- 落点：仅当前活跃终端的 TerminalView；不做会话列表落点，不做重新锚定。
- 链路：drop → `DOC_READ` 读盘上全文 → 走现有唯一粘贴通道（PASTE-1 capture-phase 路径）写入 PTY。
- 元数据块复用 focus-context 格式（与上下文注入探讨保持协议词汇统一）：

  ```text
  [Iris] 文档粘贴: .iris/issue/2026-06-12-xxx.md（类型 issue，工作区 .iris/，status: todo）
  —— 以下为拖入时快照，写回以盘上文件为准 ——
  <全文>
  ```

- 必须处理：bracketed paste（`ESC[200~ … ESC[201~`）包裹防止多行被 shell 逐行执行；剥掉内容中的 `ESC[201~` 序列防逃逸；超过 32 KiB 退化为只粘元数据 + 路径指针。
