第四批 UI/交互打磨。审查后归并为六类，按类整改。核心根因：**应用内的焦点/视图转移不触发编辑器 flush**——开终端、切集合视图、退出应用都不落盘，导致 agent 读到旧内容、甚至静默丢失。

## A. 交接给 agent 前未落盘（正确性）

核心手势是"开终端 → 注入 FOCUS\_DOC → agent cat 文档"。开终端前必须先把编辑器写盘，否则 agent 读到旧内容。

* A1 右栏开终端（`openSession`，RightPane `+` 与启动台）先 flush。

* A2 文档右键"用 X 打开"（`DocContextMenu`）先 flush（与 A1 同入口）。

* A3 右键"在资源管理器中显示"先 flush。

## B. 切走视图 / 退出前未落盘（数据安全）

* B1 点段头开集合视图（`openCollection`）先 flush。

* B2 点待办面板（`openTodos`）先 flush。

* B3 退出 / 关窗：主进程 `before-quit` 没让渲染层 flush，渲染层无 `beforeunload`——有未保存编辑时退出会静默丢失。加主进程↔渲染层 flush 握手（走正常 `doc.save` 指令，带超时兜底）。

* B4 打开"设置"视图（`openSettingsView`）先 flush。

## C. 切走后阅读位置丢失

* C1 md 阅读位置：切走再切回从头开始。按 path 记忆/恢复 scrollTop。

* C2 doc → 集合 → 回到 doc 同样丢，与 C1 同一套修法（Crepe 与源码模式各接一次）。

## D. 视觉一致性

* D1 状态灯有两套实现（LensTree 与 RightPane），还混用 ●◐○ 字形——大小/满不满都不一致。抽成共享组件 `SessionDot`：统一尺寸的纯色圆点，只用颜色分态（蓝=工作 / 黄=空闲 / 灰=退出）。

* D2 未保存标志用黄点，与终端"空闲黄"撞色撞形。改铅笔图标。

* D3 左右栏宽度 20/30 不对称。调成等宽（25/50/25）。

* D4 中栏顶部文档标题字号偏小。调大。

* D5 终端横幅显示文件名而非标题、且常被截断。改用 `docDisplayTitle` + 给标题更多空间。

## E. 终端横幅交互 + 布局

* E1 会话下拉的关闭 X 是悬停才显，看不见。改常显。

* E2 横幅无右键菜单。加"关闭会话"右键菜单。

* E3 集合/待办视图下右栏终端无意义。`ThreePane` 在 `view.kind` 为 collection/todos 时收掉右栏，让管理器铺满。

## F. 左栏增强

* F1 顶部加排序切换（修改时间 ↔ 字母序），作用于每个类型段内的文档。

* F2 顶部加文本筛选（搜文件名 + 标题）。

* F3 status / issue / report 段之间加呼吸空间。
