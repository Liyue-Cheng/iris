首先就是新建一个文档之后，焦点没有立刻切换，还要手工点击，然后就是反复点击同一篇文档，会有闪烁，我认为，重复点击同一篇文档，应该是能检测到，而不是有闪烁，你现在开始修复，搞完之后追加issue文档，然后就是点选一个复选框之后，没有立即保存

## 处理记录（2026-06-14）

三个问题各自的根因与修复：

### 1. 新建文档后焦点没有立刻切换

根因有两层：

* `doc.create` 直接 `fs.writeFile` 落盘，**没有进** **`doc.save`** **流水线**，所以
  这次写入不在 `editor-store` 的 `lastWritten` 回声去重表里。新建后
  `selectDoc` 立刻开了会话，紧接着 chokidar 的 `add` 事件又触发
  `handleDiskChange` → 因不被识别为回声而再次 `openSession`，约一个监听
  周期后把编辑器**重新挂载**（generation 自增），表现为焦点/光标被打回。

* Crepe 从不主动取焦，新建空文档后光标不在正文，需手工点一下才能写。

修复：

* `openSession` 现在在建会话时把刚加载的字节 `lastWritten.set(path, raw)`
  做种子——内容相同的即时回声（尤其是新建写入）视为零信息，直接跳过，不再
  触发重挂载；真正的外部改动字节不同，仍照常 `handleDiskChange` 重载。

* `selectDoc(path, { focusEditor })` 把取焦意图透传给 `openSession`；
  `CreateDocDialog` 新建后用 `{ focusEditor: true }` 选中，Crepe 在
  `onCreated` 里 `consumeFocusOnMount()` 后 `view.focus()`，光标直接落进正文。

### 2. 重复点击同一篇文档会闪烁

根因：`selectDoc` 对“点击当前已打开的文档”没有任何短路——重新 `DOC_READ`

* 翻 `docLoading`（loading spinner 闪一下）+ `openSession` 自增 generation
  （Crepe 重挂载），全是零信息的抖动。

修复：`selectDoc` 开头加确定性短路——`view` 已是 doc 视图、`selectedPath`
已是该文档、且无读取错误时直接返回（读失败仍允许重试）。

### 3. 点选复选框后没有立即保存

根因：在所见即所得正文里勾选任务复选框，Crepe 发 `markdownUpdated` →
`setBody` 只标脏，落盘仍要等 Ctrl+S / 失焦 / 切文档；而待办面板的 `checkTodo`
本就是立即写回的，两边行为不一致。

修复：`setBody` 里做确定性判定——若本次序列化相对上一版**只差 GFM 复选框
状态**（`[ ]`↔`[x]`，正则归一化后其余字节完全相同），即视为一次离散勾选，
立刻 `save()` 落盘；普通打字仍按原批量节奏，不受影响。

验证：`npm run typecheck`、`npm test`（47 passed）、`npm run build` 均通过。
