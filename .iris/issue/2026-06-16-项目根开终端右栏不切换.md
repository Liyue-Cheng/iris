---
title: 项目根开终端右栏不切换
status: In Review
---

在项目根打开一个终端之后，右边没有立刻切过去。

---

## 根因（agent 写回）

右栏是**锚点驱动**的：`RightPane` 只显示 `docPath` 等于中栏视图锚点的会话，
锚点 = `view.kind === 'root' ? null : selectedPath`。

而开会话的动作只把新会话设为 `activeSessionId`（`session-store.handleCreated`），
**并不切换视图**。于是当当前视图是某篇文档（选中了 doc）时，点左栏根节点旁的
**+**（`LensTree.NewRootSessionButton → openSession(null, …)`）会生成一个
`docPath === null` 的根会话——但右栏锚点还停在那篇 doc 上，新根会话被
`visibleSessions` 过滤掉，右栏不切换。

同样的耦合缺口也会出现在右键一篇**未选中**的文档去开会话时（锚点仍是旧 doc）。

## 修复

集中在 `src/renderer/lib/session-actions.ts` 的 `openSession`：开会话前先把视图
带到该会话的锚点（`docPath === null → projectStore.selectRoot()`，否则
`selectDoc(docPath)`），再 dispatch `session.open`。这样新终端必然落进右栏当前锚点、
立刻显示。导航在前、spawn 在后，`handleCreated` 的 `activeSessionId` 覆盖
`syncToRoot/syncToDoc` 选出的旧会话，新会话获得舞台。

- 不改协议、不改主进程、不加 ISA 指令，复用现有 `session.open`。
- `selectDoc` 对「已是当前 doc」会提前返回（无多余读盘/闪烁）；显式
  `flushBeforeSwitch()` 保留，保证 agent spawn 时 `cat` 到当前字节。
- 顺带统一了「右键未选中文档→用 X 打开」的行为：现在会同时选中该文档，
  符合核心手势「选中文档 → 右键 → 用 X 打开」。

涉及文件：`src/renderer/lib/session-actions.ts`。

- [ ] 右键一篇**未选中**的文档「用 X 打开」→ 中栏切到该文档、右栏立刻显示新会话。
