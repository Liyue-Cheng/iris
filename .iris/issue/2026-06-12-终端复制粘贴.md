## 调研结论（2026-06-12，对照 Marina `E:\projects\terminal`）

逐行对比了 Iris `TerminalView.tsx` / `session-manager.ts` 与 Marina 的对应实现。Iris 目前只移植了 Marina 的"骨架"（state-replay 协议、防闪烁静默窗口、8ms 聚合、headless 镜像），交互层几乎全部缺失。按严重度分组：

### 一、复制粘贴 —— 整条链路缺失（本次实现）

1. **剪贴板 IPC 通道不存在**。Marina 的教训：`navigator.clipboard` 在 Electron file:// 上下文要过 web Permission API，会静默失败；正确做法是 main 端 Electron `clipboard` 模块 + IPC 桥。Iris 的 protocol.ts 里没有任何 clipboard 通道。
2. **键位全部缺失**。Ctrl+C（有选区复制/无选区透传 SIGINT）、Ctrl+Shift+C、Ctrl+Insert、Ctrl+V / Ctrl+Shift+V / Shift+Insert 都没有 `attachCustomKeyEventHandler` 拦截。后果：Ctrl+V 被 xterm 编码成 0x16（Unix literal-next）发给 PTY，根本不触发粘贴 —— 这就是"复制粘贴没有实现"的直接根因。
3. **没有 capture-phase paste 拦截器**（Marina PASTE-1）：xterm 自带的 bubble-phase paste 监听会把剪贴板原文直接发 PTY，绕过 bracketed paste 包裹与大粘贴警告。
4. **没有右键菜单/选中即复制**。Marina：右键菜单（复制/粘贴/清屏）、selectOnCopy（100ms trailing debounce 防 Windows 剪贴板 OLE 锁抖动）、Windows 多行复制转 CRLF（CPB-C4）、复制粘贴后焦点归还终端（CPB-C1/P1）。
5. **粘贴安全检查缺失**：bracketed paste（\x1b[200~..201~，让 shell 把多行当 literal）、>1MB 大粘贴确认、内嵌 ESC（ANSI 注入）警告。

### 二、进度条/刷屏抖动 —— 根因分析（本次实现）

"进度条画花"在这套架构里有三个相互叠加的来源，全部是 Iris 漏掉的 Marina 修复：

1. **resize 无防抖**：Iris 的 ResizeObserver 每个回调立即 `fit()` 并发 IPC。拖拽分栏/窗口时一秒几十次 resize 打到 ConPTY，每次都触发 reflow 重画 —— 进度条这类 CR 重绘行最容易被撕成多行垃圾。Marina：150ms trailing debounce（用户勘误 #4）+ 最大化/还原跳过防抖立即执行。
2. **fit 无最小尺寸守卫**：布局未收敛时 fit 可能算出 cols≈0，xterm 在 0 维度下换行/光标错乱（Marina XTM-8/FLK-3，最小 20×5 才发 IPC）。
3. **spawn 用 120×30 占位尺寸**：会话先按 120 列出生、挂载后才 resize 到真实宽度。窄于 120 列的右栏里，启动横幅和早期输出按错误宽度折行，ConPTY reflow 留垃圾行（Marina 用户勘误 #2：首次 fit 后把真实 cols/rows 写回 store，后续 spawn 直接用）。
4. **打字前不冲洗待定 resize**（Marina XTM-7）：拖窗后立刻打字，PTY 还在用旧宽度处理 prompt 折行。

另：Marina/Iris 均不解析 OSC 9;4（ConEmu 任务栏进度协议），xterm 静默忽略，不是垃圾输出来源，无需处理。

### 三、切会话刷屏闪烁（本次实现）

Iris 的 replay 是一次性 `term.write(全部 scrollback)` 且不锚底：

- 大 scrollback 同步 write 阻塞主线程 100-300ms（Marina FLK-1：16KB 分片 + yield）；
- 不做 fence + `scrollToBottom()`，用户看到"从上往下刷屏"（Marina BETA-018 + SCROLL-1 第一轮：`term.write('', cb)` 是 parser drain fence，必须在 cb 里锚底）；
- 分片间 RAF 会把中间帧画上屏（SCROLL-1 第二轮：host `visibility:hidden` + `inert` 直到 fence cb + 一帧 RAF 后才 reveal，首个可见帧即终态）。

### 四、中文输入法（本次实现）

Marina IME-1/IME-2 修的是 xterm CompositionHelper 的真 bug（中文 IME 长期不按 Enter → helper-textarea 累积历史，标点 auto-convert 时整段冲刷进 PTY；候选框跟着 TUI 光标抖）。两个 workaround 是自包含模块，Iris 与 Marina 的 xterm 版本完全一致（6.1.0-beta.256），直接照搬。Iris 用户主要语言是中文，这是高价值修复。

### 五、其他缺口（本次一并实现的小项）

- **URL 不可点击**：缺 `@xterm/addon-web-links`（需新增依赖）。
- **WebGL context 丢失无回退**：Iris `loadAddon(new WebglAddon())` 后没挂 `onContextLoss` dispose（GPU 被抢占后终端白屏；Marina PER-1 模式：dispose 后 xterm 自动回退 DOM renderer）。WebGL addon 也未在 term.dispose 前显式释放（GL 句柄泄漏）。
- **自定义字体首挂度量不准**：缺 `document.fonts.ready` 后 re-fit（Marina XTM-9，霞鹜文楷场景必中）。

### 六、已调研、本次不做（记录备查）

- **终端搜索栏**（Ctrl+F + SearchAddon）：功能增强非 bug，等用户要。
- **行为设置项**（selectOnCopy / 右键模式 menu|paste / bracketedPaste 开关）：本次按 Marina 默认值硬编码（selectOnCopy=on、右键=菜单、bracketedPaste=on），要不要进设置页是产品决策，留给用户裁决。
- **输入被拒 toast**（会话已退出还在打字时的可见反馈）：Iris 尚无 toast 基建，main 端已返回 `{accepted, reason}`，UI 反馈后补。
- **退出态光标停闪**（Marina FLK-10）：Iris 未开 cursorBlink，不存在此问题。
- **SSH DA 响应过滤 / 多窗口 ownership / OSC cwd 跟踪**：Marina 特有，Iris 架构已裁掉，无需移植。

### 与 Marina 的两处有意偏差

1. **粘贴用 `term.paste()` 而不是手工包 `\x1b[200~..201~`**：xterm 会按运行中程序是否真开了 mode 2004 决定是否包裹（PSReadLine / agent CLI 都开）。免去 bracketedPaste 设置项，cmd.exe 下也不会出现字面 marker；mode 关闭时多行粘贴仍走确认弹窗兜底。
2. **确认弹窗用原生 `window.confirm`**（>1MB 大粘贴 / 含 ESC / 非 bracketed 多行）：Iris 还没有 Modal 基建，等有了再换。

另修一个 Marina 也没有的隐性 bug：原 `btoa(String.fromCharCode(...bytes))` 在大输入（如 1MB 粘贴经 onData 回流）会爆参数栈，改分块编码。
