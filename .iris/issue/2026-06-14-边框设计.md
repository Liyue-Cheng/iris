现在软件有很多明暗不一的边框，有一些不必要、有一些该有却没有。对标 TRAE
（SOLO Coder）重新梳理边框语言。

## 核心原则（从 TRAE 截图归纳）

> 边框只用来表达「这是一个可交互的容器」；结构分区一律靠背景明度差，不靠线。

TRAE 截图证据：

- 三栏之间几乎看不到分隔线，只有一条极淡竖发丝，主要靠右侧终端区背景略浅分区。
- 横向 `border-b` 头条基本不存在——「新任务」「选择项目」标题栏、终端 tab 栏
  下方都没有下边框，头与正文靠背景/留白分开。
- 有边框的地方全是「你要去点的东西」：底部输入框（圆角卡片 + 淡边框 + 内部头
  分隔）、「新任务」按钮、「实时跟随」激活态（绿色 pill 边框）、终端 tab。
- 激活/选中 = 上色边框；静止态边框极淡。

Iris 现状恰好相反：用边框做「结构」（每个 pane 头都 `border-b`），又把它们减弱
成 15%~38% 乱跳的泥；真正该有卡片感的交互件反而和结构线同权重，主次不分。

## 诊断（明暗不一的根因）

token 层已克制：`--border` 本身只有 38% 不透明度
（`color-mix(--rp-highlight-med 38%, transparent)`，global.css:61）。问题在用法：

| 用法 | 出现位置 | 实际有效 alpha |
| --- | --- | --- |
| `border-b`（裸） | TitleBar、各 Pane 头、collection 头、TypedHeader | 38% |
| `border-border/40` | 设置项分隔、表格行、IssuePanel 行 | ≈15% |
| `border-border/60` | LensTree 缩进线、dashed 按钮 | ≈23% |

即：在一个已是 38% 的半透明 token 上各处再乘 /40、/60，最终亮度乱跳，没有统一
刻度——这就是「明暗不一」。

## 改造方案

### 删（结构线 → 改背景差，头条走纯背景差，已定）

- TitleBar.tsx:182 的 `border-b`（标题栏已有 `bg-card`，靠它分区）
- 各 pane 头 `border-b`：LeftPane.tsx:62 / 180、RightPane.tsx:68
- collection/doc 头 `border-b bg-card/30`：IssuePanel.tsx:158、SimpleList.tsx:39、
  TodoPanel.tsx:82、RootReadme.tsx:70、TypedHeader.tsx:242（保留 `bg-card/30`
  背景差，去掉线）
- SettingsView.tsx:144 / 163 的 header / nav 边框同理

### 删（违和）

- 所有 dashed：SettingsView.tsx:247 / 599 / 916、TypedHeader.tsx:204
  → 换 hover 显形的实线或纯文字

### 保留（TRAE 认可的「交互容器」边框）

- `ResizableHandle`（静止淡、hover 显形，resizable.tsx:29）——完全对路，不动
- input / button outline / dialog / dropdown / context-menu / tooltip —— shadcn
  那套卡片边框，留着

### 改（二次稀释 → 两档制）

全项目只允许两档边框亮度：

- `--border`（结构性，基本只剩分隔条用）
- 新增 `--border-subtle`（列表行/缩进线类，固定一值，约 20%）

改用 `--border-subtle` 的位置：IssuePanel.tsx:230、SettingsView.tsx:230 / 544 /
867、LensTree.tsx:210——不再各写各的透明度。

## 决策记录

- 横向头条去 `border-b` 后，**走纯背景差**（最贴 TRAE），不留发丝线。
  - 风险：Rose Pine 暗色下 `bg-card/30` 分区可能偏弱；实现后需在三变体逐一目检，
    若过弱再考虑加大头条背景差（如头 `bg-card` 实色 + 内容区 `bg-background`）。
- 设置区左侧 nav 是**竖向**分隔，与 `ResizableHandle`（保留的极淡竖线）同类，
  不归入「横向头条」的纯背景差规则——降级为 `border-subtle`，保留一条极淡竖线，
  而非完全删除。

## 实现记录（typecheck 通过，待人工目检）

token 两档制（global.css 三变体 + tailwind.config.cjs）：

- `--border` 38%（结构性 / 卡片）；新增 `--border-subtle` 20%（列表行 / 缩进线）。
- tailwind 注册 `subtle` 颜色 → 类名 `border-subtle`，全项目第二档唯一来源。

删头条 `border-b`（保留 `bg-card/30` 背景差）：TitleBar、LeftPane（头 + 筛选行）、
RightPane、IssuePanel、SimpleList、TodoPanel、RootReadme、TypedHeader、
SettingsView header。

删 dashed（→ `border-subtle` 实线、hover 显形）：TypedHeader 加标签按钮、
SettingsView Placeholder、两处 `+ chip` 快速添加按钮。

二次稀释 `/40`、`/60` → `border-subtle`：IssuePanel 行、SettingsView Field/
模板表/agents 表、LensTree 缩进线。

保留不动（TRAE 认可的交互容器边框）：`ResizableHandle`、input / button outline /
dialog / dropdown / context-menu / tooltip。
