## 产品是什么

Iris 是一个 AI 原生、文档中心、终端驱动的项目管理工具（Windows 桌面端，Electron）。
它的世界观：所有项目数据是磁盘上的 markdown 文件（文件系统即数据库），所有智能来自
用户自带的 agent CLI（claude / codex 等，跑在内嵌终端里）。应用本体只是"查看器 +
召唤器"——一层套在 markdown 文件和终端会话池上的最薄外壳。

核心手势：在文档树里选中一篇文档 → 右键 → "用 claude 打开" → 右栏弹出新终端，
agent 已知道聚焦的是哪篇文档。人的工作 = 在文档间巡航，把 agent 派遣到文档上，
回来收割结果。

气质关键词：安静、克制、文字优先、低镀铬（low-chrome）、注意力调度台。
参照系：Linear 的密度与秩序感 + Obsidian 的文档氛围 + 终端的素朴。
反面教材：花哨的 dashboard、大圆角大留白的营销风、拟物。

## 设计语言（硬约束，已有设计系统项目「Iris 设计系统」可关联）

- 色板：Rose Pine 三变体（rose-pine 默认深色 / moon 中度暗紫 / dawn 浅色），
  双层 token——组件只允许消费 shadcn 语义 token（--background/--card/--primary 等），
  禁止出现 token 之外的颜色字面量。每屏至少给 rose-pine 深色稿，关键屏补 dawn 浅色稿。
- 语义色惯例：foam(#9ccfd8)=进行中、gold(#f6c177)=等待/未保存/例外、
  pine(#31748f)=完成、love(#eb6f92)=阻塞/危险、iris(#c4a7e7)=品牌/主操作。
- 字体：UI 与正文 LXGW WenKai（霞鹜文楷），代码与终端 Cascadia Mono。
- 密度：正文 14px/1.75，列表行 13px，辅助文字 11–12px，标题层级克制
  （文档 h1 仅 1.5em）。圆角 0.5rem 封顶，阴影只给弹层。
- 窗口：frameless 自定义标题栏（拖拽区 + 最小化/最大化/关闭），整体三栏布局，
  分隔条可拖动。界面语言中文。

## 信息架构（三栏）

左栏 = 透镜树（注意力调度面板）：按工作区分组 → 组内按类型分组
（status/issue/report/misc）→ 文档行。每行左侧 12px 会话状态点：
● foam=agent 工作中、◐ gold=空闲等输入、○ muted=已退出、空=无会话。
issue 行尾随 status 徽章。归档内容 60% 透明度灰化。底部有"裸文件树"逃生舱 toggle。

中栏 = 文档区：单篇视图（类型化头部：类型徽章 + title + status 下拉 + 脏状态点，
正文是 Typora 式所见即所得编辑器，可切源码模式）；集合视图（点类型标题进入——
issue 是 Linear 风格列表：过滤药丸"活动中/已解决/全部"带计数、状态徽章原位下拉、
归档行灰化；其余类型是简单列表）。

右栏 = 终端池：xterm.js 终端 + 会话列表（每会话一行：agent 名、锚定文档、状态点），
可切换，同一文档可挂多个会话。
