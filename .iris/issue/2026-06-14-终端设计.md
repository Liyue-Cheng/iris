## 全面对比结论（Iris 终端 vs Marina 终端）

逐文件比对了两边的 TerminalView、主进程窗口创建、键位表、主题表。Marina TerminalView 2191 行 / Iris 617 行，差异集中在以下几处。

### 你报告的三个问题 —— 均已定位根因并修复

1. **打开链接无效**
   - 根因：Iris 主进程**完全没有** `setWindowOpenHandler`。WebLinksAddon / OSC 8 点击触发 `window.open`，被 Electron 默认拦截后直接丢弃。
   - 修复：`src/main/index.ts` 加 `setWindowOpenHandler`，白名单 `http/https/mailto` → `shell.openExternal`，其余 deny（防 OSC 8 注入打开本地文件）。对齐 Marina `window-manager.ts:378`。

2. **拖入文件粘贴无效**
   - 根因：Iris 的 `onDrop` 只处理内部文档拖拽（`getDocDragPath`），从不读 `dataTransfer.files`，所以从资源管理器拖文件进终端无反应。
   - 修复：`TerminalView` 新增 `handleFileDrop` + onDrop 的 OS 文件分支：取 `file.path`、含空格加引号、shell 元字符弹 confirm，直接走 `SESSION_INPUT` 发给 PTY（不走 paste —— 拖入的是命令参数，不是剪贴板内容）。Electron 31 仍提供 `file.path`，32+ 再迁 `webUtils`。onDragOver 同步识别 `Files` 类型显示 copy 光标。

3. **浅色主题文字太淡/看不见**
   - 根因：Iris 的 `Terminal` 构造**没设** `minimumContrastRatio`。主题表虽已带 `extendedAnsi`，但缺这个 WCAG 兜底开关，CLI 的 dim 字（`\x1b[38;5;245m`）在浅底上对比度 <3:1。
   - 修复：`xterm-themes.ts` 补 `isLightTheme()` + `LIGHT_THEME_MIN_CONTRAST=4.5`；`TerminalView` 构造与运行时主题 effect 都设 `minimumContrastRatio: isLightTheme ? 4.5 : 1`（深色主题保持 1，不破坏故意调淡的层级）。

### 全面检查另发现并迁移的功能（你已确认要做）

4. **搜索（Ctrl+F 搜索栏）** —— Iris 原先完全没有。已迁移：装 `@xterm/addon-search`；键位表补 `open-search`/`close-search`（Esc 带 `searchVisible` guard，关栏时透传 PTY）；`matchKeybinding` 加 context 参数；搜索栏 UI（命中数 x/N、上/下一个、区分大小写、关闭）走 Tailwind + shadcn token；`onDidChangeResults` 喂命中计数；输入即搜 + Enter/Shift+Enter 跳转。单测补 Ctrl+F / Esc 用例。

5. **Ctrl+滚轮调字号** —— Iris 原先没有。已迁移：host onWheel，Ctrl/Cmd+滚轮在 8–24px 间调，本地即时生效 + trailing 120ms debounce 经 `settings.update` 写回。

### 判定为不适用 Iris、不迁移的 Marina 终端特性

SSH 重连按钮、SSH 启动期 DA 响应过滤、多窗口 owner 占位、终端状态栏（pid/cwd/exitCode/简易模式切换）、输入被拒 toast、IME 调试探针 ring、i18n —— 均属 Marina 特有产品形态，Iris 不需要。

`npm run typecheck` 干净；`npm run build` exit 0；全量单测 47 passed（含新增搜索键位用例）。

## 追加：右栏标题栏改为实时终端标题（OSC 0/2）

### 问题
右栏标题栏原本显示「agent 标签 + 文档标题」，是静态文档信息，不是终端标题，也不随程序运行实时变化。

### 根因
Iris 的 `SessionInfo` 没有终端标题字段，主进程也没捕获 OSC 0/2 标题序列。Marina 在主进程用自研 OSC parser 捕获标题并更新 `displayName`。

### 修法（与 Marina 一致，但复用 Iris 已有的 headless 镜像）
- `SessionInfo` 新增 `terminalTitle: string | null`（实时终端标题；保留 `displayName` 作稳定 agent 标识，供下拉区分多会话）。
- 主进程在每个 session 的 `headlessTerm` 上挂 `onTitleChange`（headless 镜像本就逐字节解析全部输出，OSC 0/2 会触发它，无需另写 parser），经 `handleTitle` 规范化 + 启动垃圾过滤 + 去重后 `emitStateChanged({ terminalTitle })` 广播。
- 移植 Marina 的 `sanitizeTitle`（去控制/双向重写字符、合并空格、截断 100）与 `looksLikeShellStartupGarbage`（滤掉 ConPTY 把 shell 启动期 SetConsoleTitle(exe 路径) 翻成的 OSC 0、Git Bash 每个 prompt 重发的 `MINGW64:<cwd>` 等）。
- 广播链已通：`sessionStateChanged → ipc → interrupts → sessionStore.handlePatch`（patch 全量合并，无键过滤），React 自动重渲染。
- `RightPane` 标题栏主标题改显 `terminalTitle ?? displayName`，agent 标签退为尾部 muted 小标签；下拉每个会话也带实时标题。文档锚点信息保留在「+/下拉」label 里。

### 验证
`npm run typecheck` 干净；全量单测 47 passed。

### ✋ 待你手工验收（追加）

- shell 刚启动时不应把标题刷成 `C:\…pwsh.exe` 之类启动垃圾。
- 同一文档挂多个会话，下拉里各自显示自己的实时标题。

## 追加：标题栏精简 + 下拉自适应宽度

### 改动
1. **移除标题栏尾随 agent 标签** —— 右栏标题栏原本在实时终端标题后还跟一个 muted 的 agent 名（如 `claude`），属冗余。已删除 `RightPane` 中 `terminalTitle && <span>{displayName}</span>` 块，标题栏现在只显示 `terminalTitle ?? displayName`。agent 身份仍保留在下拉里，多会话照样可区分。

2. **下拉宽度改为最小长度 + 自适应扩展** —— 原 `DropdownMenuContent` 固定 `w-64`，长标题被截断。改为 `w-max min-w-64 max-w-[32rem]`：按内容自适应、保底 16rem、最长到 32rem 才触发省略号。会话项与 label 仍带 `truncate`，仅触顶时截断。
