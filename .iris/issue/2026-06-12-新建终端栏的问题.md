## 调研（agent 写回，待审核）

### 现状：Iris 右栏的三个新建入口

全在 `src/renderer/components/layout/RightPane.tsx`：

1. **顶栏 + 按钮 → 下拉菜单**：纯文本菜单项（「用 claude 打开」「终端」），无图标、无视觉层次。
2. **选中文档无会话时的启动面板**：一行灰字 + 一列 192px 宽的 secondary 小按钮，竖排堆叠，只有文字。
3. **无文档选中时**：两行灰色提示文案，**没有任何可点击入口**——想开项目根会话必须自己发现右上角那个很小的 +。

### Marina 的对应实现（`E:\projects\terminal`）

新建终端是一个有视觉身份的一等页面 EmptyPathState（`MainPane.tsx:217-379`、`global.css:1461-1543`）：

- **整页居中布局**：max-width 720px、padding 32px 48px、分组间 gap 20px。
- **顶部 hint 行**：「在 `<path>` 新建终端」，路径用 monospace 字体 + gold 色 + elevated 底色块。
- **分组标题**：11px 大写、letter-spacing 0.6px、次级灰。
- **按钮网格**：`grid auto-fill minmax(160px, 1fr)`，gap 8px，随面板宽度响应式排列。
- **模板按钮**：图标（16px，iris 紫）+ 标签，bg-secondary 底、1px 边框、圆角 6px、padding 10×12；hover 时背景提亮、**边框亮成 iris 紫**，transition 80ms。
- **图标系统**：TemplateIcon 组件，内置模板映射矢量图标，自定义模板用 emoji。
- **无路径选中时**：WelcomeState 欢迎页（大标题 + 引导句）。

**差距一句话**：Marina 把「新建」当作产品门面来做；Iris 现在只有兜底文案级别的实现。

## 整改方案

**范围**：纯 renderer UI 改造，不动协议、不动主进程、不加 ISA 指令（复用现有 `session.open`）。

### 1. 启动面板重做（核心）

新组件 `src/renderer/components/terminal/SessionLauncher.tsx`，从 RightPane 中拆出空态逻辑，按 Marina EmptyPathState 风格重做：

- 居中列布局：右栏比 Marina 主区窄，取 max-width ~360px、padding 24px、分组 gap 20px。
- **选中文档时**：
  - hint 行：挂在 `<文档名>` 的新会话——文档名 monospace + gold 色 + elevated 底色块（与 Marina 同款处理）；
  - 分组「用 X 打开」：agent 按钮网格 `repeat(auto-fill, minmax(120px, 1fr))`，gap 8px；
  - 分组「项目根（无聚焦兜底）」：同款网格，视觉权重略低。
- **无文档选中时（欢迎态）**：引导句 +「项目根」agent 网格直接给出——消灭「必须去找 + 菜单」的死角。
- 按钮样式：图标 + 标签，`bg-secondary`、1px 边框、圆角 6px；hover 背景提亮、边框变 `--rp-iris`、transition 80ms。颜色全部走主题 token，三变体（rose-pine / moon / dawn）通用。

### 2. agent 图标

`AgentConfig` 目前只有 `{ id, label, command }`，没有图标。两个选项：

- **方案 A（推荐）**：渲染层内置 id → lucide 图标查表（`claude` → Sparkles、`codex` → Bot、`command` 为空的裸 shell → SquareTerminal），未知 id 兜底 SquareTerminal。零 settings schema 改动，壳保持哑。这是字面 id 查表，不是启发式，不碰确定性红线。
- 方案 B：`AgentConfig` 加可选 `icon?: string`（emoji），settings.json 里配。更可定制，但 v1 没有设置 UI，徒增一处 schema。

### 3. + 下拉菜单

保留（它是已有会话时的快捷路径），菜单项加上与启动面板同款图标，分组结构不变。

### 4. 不动的部分

- 会话列表行与状态点（M3 已落地，非本 issue 范围）；
- 锚定模型、`session.open` 指令、终端本体。

### 涉及文件

- 新增 `src/renderer/components/terminal/SessionLauncher.tsx`（含 agent 图标查表）；
- 修改 `src/renderer/components/layout/RightPane.tsx`（空态替换为 SessionLauncher，+ 菜单加图标）；
- `src/renderer/styles/global.css` 仅在 Tailwind 组合不够时少量补充。

### 验收点

1. 选中无会话文档 → 启动面板出现 hint 行 + 两个分组网格，按钮带图标，hover 边框变 iris 紫；
2. 无文档选中 → 欢迎态可直接开项目根会话；
3. + 菜单项带图标，行为不变；
4. 三个主题变体下视觉正常；
5. `npm run typecheck` 通过。
