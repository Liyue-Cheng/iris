## 需求理解（2026-06-12，Claude 整理；第二批补充后已修订）

这一轮的主线是四件事：**上下文注入落地**（misc 探讨文档从调研转实现）、**状态与标签体系正式化**（issue / report / 标签三套规约 + 可配置渲染 + 规约文本跟随修订）、**三栏壳的视觉整改**（顶栏重排 + 整体放大 + 联动收尾 + 第二轮遗留）、**删除文件手势**（含终端善后）。外加一条哲学修订。逐条展开：

### A. Agent 设置 + 上下文注入（实现 misc/探讨：上下文注入和联动机制 的全部结论）

设置页新增完整的 Agent 设置区，把探讨文档的方案落成产品：

* 每个 agent 一条配置：名称、启动命令、注入通道（SessionStart hook / 启动 flag / 无——降级回 AGENTS.md 引导）。

* 机器级 `~/.iris/` 落一个 focus-context 脚本（读 `FOCUS_DOC` + cwd，输出元数据块 + 文档全文快照），五家 hook 共用这一个脚本。

* hook 配置住在用户自己的 agent 配置文件里，Iris 在设置页里检测、建议、**经用户确认后代写**——与 AGENTS.md 引导段同一姿势，壳保持哑。

* 协议层不动：壳依然只设 `FOCUS_DOC`。

探讨文档里三个待拍板项，**已照准**：① 定义书"裸启动"措辞修订为"不传任务 prompt，但可经用户配置注入上下文"——与本轮 G 条哲学修订一并改；② 注入默认带全文，超 32 KiB 退化为只注指针；③ 分发形态走"设置页检测 + 经确认代写"，不只出文档。

### B. Issue 状态规约正式化

* **已拍板**：默认状态集为六态，**存盘值即显示值**：`Todo` / `In Progress` / `In Review` / `Blocked` / `Done` / `Canceled`。不设小写规范值、不加显示名映射层——标签直接原样渲染字段值，与"键按字面解析"的渲染红线同构。

* 状态标签渲染：一张可配置对应表（状态字符串 → 标签样式，字面精确匹配）；未配置的状态渲染为灰色默认标签——"键是硬的，值是软的"，自由字符串状态依然合法，写错大小写也只是落灰色默认，属优雅降级而非错误。

* 存量迁移：盘上已有 issue 的 `todo` / `in_progress` 等小写值做一次性扫描改写为新值。

* 规约文本随之修订，见 H 条。

### C. Report 状态规约

* report 只有两态，同样存盘值即显示值：`Active` / `Backlog`；`Backlog` 的 report 在左栏不显示。

* 与 append-only 的张力随 H 条规约修订一并解决：明确为"正文 append-only，frontmatter 状态流转允许"（与 issue 的"不删除、改 status 标记解决"同一逻辑）。

### D. 标签（labels）样式配置

* 内置多种标签样式；一张可配置对应表：标签字符串 → 样式。

* 未配置的标签同样落灰色默认。与 B 的状态样式表是同构的两张表。

* **已拍板（配置归属）**：项目级优先；初始化项目时把机器级默认配置写入项目作为起点，之后项目内自行演化。落点遵守红线：项目级住 `.iris/`、默认值住 `~/.iris/`（具体文件名实施时定）。

### E. 三栏联动与布局整改

1. **右栏只显示与中栏文档对应的终端**（第二轮方案 3 的最终形态：过滤式，不是高亮式）。
2. **右栏顶栏**：既然只剩当前文档的终端，去掉会话 tab 条和关闭 ×，换成一个终端 banner——点击可切换该文档下的终端、右侧可新建。
3. **中栏顶栏定为两行**：第一行 状态 + 标题，最右保存状态标记 + 代码按钮；第二行 优先级 + 标签列表。路径不常驻，悬浮标题时显示；去掉 issue/report 大类型标签。
4. **左栏根节点特殊化**：根目录不再是点击折叠的普通节点——点击后中栏显示项目 README（无则显示"暂无 README"占位），右栏显示项目根终端。
5. **顶栏第一行对齐**：左中右三栏顶栏的第一行彼此等高对齐；左栏小按钮随之对齐。
6. **边框减弱**：当前分栏边框太重，向 VS Code 的克制程度看齐（Rose Pine token 层调整）。
7. **整体放大**：界面太小、不够大气，作为编程工具要整体放大一档（基准字号 / 间距 / 行高一起动，而不是只调 uiZoom 默认值——具体方案实施时给你过目）。

### F. 第二轮遗留

1. **Marina 式 EmptyPathState 整页整改**：第二轮只落了文档挂靠启动面板，整页空态未做，本轮补完。
2. **Portable 版 GPU cache 创建失败报错**：疑似多实例共享 userData 或目录权限，需定位并修复（方向：portable 模式下 userData 隔离 / `--disk-cache-dir` 显式指定）。

### G. 设计哲学修订

"本程序不提供 AI"的红线措辞需要松绑：坚持的是**不内嵌 agent、不做订阅、智能来自用户自带 CLI**；放行的是**用户 BYOK 驱动的辅助功能**（如自动填标签）。本轮先改 software-definition.md 的哲学表述（与 A-① 的"裸启动"措辞修订同一次改动），具体 BYOK 功能不在本轮实现范围。

### H. 规约修订（CONVENTIONS 跟随迭代）

**已拍板原则**：软件迭代，规约跟着改，合理。本轮规约层面的改动汇总：

* issue 状态机改为六态，存盘值即显示值（B 条）；report 两态同理（C 条）。

* report 明确"正文 append-only、frontmatter 状态流转允许"（C 条）。

* 存量文件的小写状态值一次性迁移为新值（B 条）。

* 落点有两处：`iris-templates.ts` 的模板（管未来新项目，我直接改）；本项目已落盘的 `.iris/CONVENTIONS.md`——**已获你明确授权**，实施 H 条时由我同步修改（仅限本轮拍板的规约改动，不扩大范围）。

### I. 删除文件手势

* 文件右键菜单补"删除文件"。

* 删除时若有终端挂载在该文档上：弹窗确认，给两个选择——直接关掉这些终端，或把终端**重新锚定到项目根**（已拍板；Marina 锚定模型本来就支持"路径改文档"，复用即可）。

* 无终端挂载时是否也弹普通删除确认：建议弹（删除不可逆），实施时按此做。

* 注：CONVENTIONS 的"issue 不删除、改 status"约束的是 agent 的写回行为；用户在 UI 里的手动删除是另一回事，不冲突。

### J. md 编辑器右键菜单

* 中栏 Crepe 编辑器区域目前完全没有右键菜单，补上。

* 至少覆盖标准编辑操作（剪切 / 复制 / 粘贴 / 全选），样式与既有的共享右键菜单（左栏 / issue 列表那套）保持一致；编辑器特有的项（如块操作）实施时再定，不在本条预设。
