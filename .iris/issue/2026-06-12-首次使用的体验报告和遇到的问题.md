## 工作记录（2026-06-12，agent）

逐条处理结果：

* **持久化位置 / 打包版与 npm 版互踩**：现在的持久化只有一处 `~/.iris/settings.json`。移植了 Marina 的 build-type 探测（dev / portable / installed），dev 构建改存 `~/.iris-dev/settings.json`，打包版（portable 与 installed）仍用 `~/.iris/`；`~/.iris/CONVENTIONS.md` 不分叉——机器事实与构建形态无关。注意：npm 版下次启动会回到默认设置（旧 `~/.iris/settings.json` 归打包版所有）。

* **F12 / 独立开发者工具**：打包版没有应用菜单，Chromium 不会自动绑定 F12，需主进程拦截 `before-input-event`（Marina 的做法）。现在 F12 与 Ctrl+Shift+I 在 dev 和打包版都可切换 DevTools，一律以独立窗口（detach）打开；另加 `IRIS_DEVTOOLS=always` 环境变量供诊断打包版启动期错误。

* **默认标题栏**：照抄 Marina M1-A——`frame: false` 移除系统标题栏，TitleBar 整条作为拖拽区（双击最大化为原生行为），右侧新增最小化 / 最大化（还原）/ 关闭按钮，走 `window:*` IPC；最大化状态由主进程广播，按钮图标随之切换。

* **右键菜单**：左栏 LensTree 此前已有"用 X 打开"，缺的是其余面。抽出共用的 DocContextMenu（按 settings.agents 逐项"用 X 打开" + "在资源管理器中显示"），接入 issue 面板、简单列表与原始文件树的 md 行；`shell:reveal` 支持项目内相对路径。

* **标题重复 / 新建 issue 字号过大**：两者同根——脚手架在正文里重复写了 `# 标题`，被 Crepe 按一级标题渲染。决议：frontmatter `title:` 保留为唯一声明源（类型化头部拥有并显示它，符合"frontmatter 不进正文编辑器"），新建文档不再写正文 H1；无 title 的文档按既有逻辑回退为文件名显示。不采用"从正文启发式取标题"：`status:` 等字段反正需要 frontmatter，且砍掉 title 会让头部改标题时不得不对正文动手术。

* **设置页面**：未做。后端（SettingsManager：局部更新 / 校验 / 广播 / 即改即生效）已齐备，缺的只是 UI；按"一次一个里程碑"纪律应作为独立条目排进 roadmap。标题字阶等排版自定义依赖它，本次未实现。

验证：`npm run typecheck` 通过；29 个单测全部通过；build + smoke 通过（dev 形态 settings 已落到 `~/.iris-dev/`）。

手工验收建议：① 窗口无系统标题栏，可拖拽、双击最大化，右上三键可用；② F12 弹出独立 DevTools 窗口（dev 与打包版都试）；③ 三个列表面与原始树的右键菜单出现，"用 claude 打开"正常注入 FOCUS\_DOC；④ 新建 issue 后正文无巨大标题；⑤ npm 版与打包版的设置互不影响。
