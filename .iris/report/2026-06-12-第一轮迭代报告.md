# 第一轮迭代报告（dogfood 第 1 轮）

时间：2026-06-12。背景：M5 + portable 打包（`1fb7f9f`）之后，首次在打包版上真实使用本项目管理自身，本轮共产生 10 个 issue、1 篇 misc 探讨，修复与新功能以 7 个提交落地（`636ae99`..`faadbfa`），typecheck 全绿、43 个单测全过。

## 本轮交付

### 1. 应用壳层补课（`636ae99`）

来源：issue「首次使用的体验报告和遇到的问题」「MD（块手柄）」。

- 无边框窗口 + 自绘标题栏（最小化/最大化/关闭走新的 window:\* IPC），照抄 Marina。
- 设置页（视图而非弹窗）落地，设置 schema 扩了 behavior 组（选中即复制、终端右键行为、退出确认、编辑器 BlockEdit 开关）、界面缩放、终端行高、终端渲染器选择，全部带校验。
- 持久化分叉：dev 构建写 `~/.iris-dev`，打包版写 `~/.iris`，dev 与打包版互不践踏（Marina DEV-COEXIST 教训）。
- F12/Ctrl+Shift+I 在打包版可开 DevTools，且为独立窗口；`IRIS_DEVTOOLS=always` 作为打包版启动期诊断后门。
- 全局滚动条主题化（Rose Pine token）+ `scrollbar-gutter: stable both-edges` 根治编辑器正文列随滚动条出现/消失的水平抖动。
- Crepe 块手柄（BlockEdit）做成设置项并默认关闭——它每换悬停块就重定位一次，视觉上是频繁闪动。

### 2. 会话投影修复 + 文档↔会话联动（`c23af7c`）

来源：issue「会话投影丢失」（已 done）、「第二轮需要处理的问题」第 3 条。

- 根因：session-store 只靠增量事件维护、无全量恢复路径，渲染端一重载投影即清空而 PTY 池仍在。修复：启动与项目打开时从主进程拉全量快照（hydrateSessions）；ISR 遇到未知 sessionId 视为确定性失步证据，自动重拉。
- 联动：左栏选中文档 → 右栏自动切到该文档下最优会话（active > idle > exited），没有会话则显示挂靠该文档的启动面板；会话列表按锚点过滤，销毁回退优先同锚点兄弟会话。

### 3. 终端交互加固——Marina 移植（`d010cc7`）

来源：issue「终端复制粘贴」「新建终端栏的问题」（部分）。

- 复制粘贴整条链路：剪贴板 IPC 桥（navigator.clipboard 在 file:// 上下文静默失败）、共享键位表（Ctrl+C 有选区复制/无选区 SIGINT、Ctrl+Shift+C/V、Shift+Insert，带单测）、capture-phase 单一粘贴入口、右键菜单/直接粘贴二选一、选中即复制（防抖）、Windows CRLF 转换。
- 粘贴安全：>1MB 预检、内嵌 ESC（ANSI 注入）警告、非 bracketed-paste 程序的多行确认。
- IME-1/IME-2 两个 xterm 中文输入法 bug 的运行时补丁（积压历史冲入 PTY、候选框抖动）。
- resize 纪律：150ms 尾沿防抖、<20×5 不通知 PTY、按键前冲刷待定 fit、真实尺寸写回使新会话按正确尺寸 spawn（ConPTY 对 spawn 后立刻 resize 会把早期输出画花）。
- 重放防闪烁（先锚定再显示）、WebGL 上下文丢失自动降级 + 渲染器设置、web-links、xterm 滚动条与全局观感对齐。

### 4. 文档右键菜单与拖拽（`dde4530`）

来源：issue「第二轮需要处理的问题」第 7 条。统一的 DocContextMenu（用 X 打开 + 在资源管理器中显示）覆盖透镜树、原始树、列表行；同一批行成为拖拽源，拖进终端即粘贴元数据头 + 全文快照（>32KiB 改贴指针）。

### 5. 标签、优先级与待办面板（`ae20d93`）

来源：issue「新功能：代办」。

- 扫描器新增两个字面投影：frontmatter `labels:`（无注册表，标签集合即全项目实际使用的并集，颜色为名字的确定性哈希）与正文 GFM 任务项（逐行解析、确定性围栏追踪，含单测与夹具）。
- issue 面板新增优先级列（urgent/high/medium/low 软值，内联下拉，参与排序）、标签列（点击即过滤）；正文头部可增删标签，写回为单行 YAML flow sequence，沿用行级手术纪律。
- 待办面板：活动 issue 的全部未勾选任务，按来源分组，左栏入口带计数徽章；勾选是单行字节手术，写前与扫描时的原始行做字节比对，不一致即拒绝、等下一次扫描。
- 状态下拉从 datalist 换成真菜单（datalist 按当前值过滤候选，已是 done 的字段什么都选不了——issue「第二轮」第 2 条）。

### 6. 协议修正（`b83faec`）

来源：issue「第二轮需要处理的问题」第 6 条。CONVENTIONS/AGENTS 模板删掉"每次任务后追加 report"的规则（它在诱导 agent 乱建文件），换成显式的"未经要求不新建文件"；新建文档脚手架去掉正文 H1（标题由类型头部呈现，原来渲染两遍）。
